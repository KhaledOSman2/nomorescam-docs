# التقرير التقني لمشروع No More Scam

هذا المستند يحتوي على التفاصيل التقنية الكاملة لمشروع "No More Scam"، وهو منصة مجتمعية تهدف إلى مكافحة الاحتيال الإلكتروني من خلال تمكين المستخدمين من الإبلاغ عن المحتالين والبحث عنهم. تم بناء المشروع باستخدام أحدث التقنيات لضمان الأداء العالي، الأمان، وتجربة المستخدم السلسة.

---

## 1. نظرة عامة على المعمارية (Architecture Overview)

يعتمد المشروع على معمارية **Client-Server** حديثة ومنفصلة (Decoupled Architecture):
- **الواجهة الأمامية (Frontend):** تطبيق صفحة واحدة (SPA) مبني باستخدام React.
- **الواجهة الخلفية (Backend):** واجهة برمجية (RESTful API) مبنية باستخدام Node.js و Express.
- **قاعدة البيانات (Database):** قاعدة بيانات غير علائقية (NoSQL) باستخدام MongoDB.

---

## 2. الواجهة الأمامية (Frontend)

تم تصميم الواجهة الأمامية لتكون سريعة، متجاوبة، وسهلة الاستخدام.

### التقنيات المستخدمة:
- **React + Vite:** لبناء واجهة المستخدم بسرعة أداء عالية وتجربة تطوير ممتازة.
- **TypeScript:** لضمان سلامة الكود (Type Safety) وتقليل الأخطاء أثناء التطوير.
- **Tailwind CSS:** لتصميم الواجهات بشكل سريع ومرن (Utility-first CSS framework)، مع ضمان التجاوب مع جميع أحجام الشاشات.
- **Zustand:** لإدارة حالة التطبيق (State Management) بشكل بسيط وفعال (مثل حالة تسجيل الدخول).
- **React Router DOM:** لإدارة التنقل بين الصفحات (Routing) داخل التطبيق.
- **React Hook Form + Zod:** لإدارة النماذج (Forms) والتحقق من صحة البيانات (Validation) بشكل قوي ومرن.
- **Axios:** لإجراء طلبات HTTP والتواصل مع الـ Backend.
- **React Helmet Async:** لإدارة الـ SEO وعلامات الرأس (Meta Tags) لكل صفحة.
- **Lucide React:** مكتبة أيقونات خفيفة وعصرية.

### المميزات التقنية في الواجهة الأمامية:
- **تصميم متجاوب (Responsive Design):** يعمل بكفاءة على الهواتف، الأجهزة اللوحية، والحواسيب.
- **حماية النماذج:** استخدام Cloudflare Turnstile (أو مشابه) لحماية النماذج من الروبوتات (Bots).
- **تجربة مستخدم تفاعلية:** استخدام التنبيهات (Toasts) والتحميلات (Loaders) لتحسين تجربة المستخدم.
- **Code Splitting:** تقسيم الكود لتحميل الصفحات المطلوبة فقط، مما يزيد من سرعة الموقع.

---

## 3. الواجهة الخلفية (Backend)

تم بناء الواجهة الخلفية لتكون آمنة، قابلة للتوسع، وسهلة الصيانة.

### التقنيات المستخدمة:
- **Node.js + Express:** بيئة التشغيل وإطار العمل لبناء الـ API.
- **TypeScript:** لكتابة كود backend قوي ومنظم.
- **MongoDB + Mongoose:** قاعدة البيانات ونظام النمذجة (ODM) لتخزين البيانات بمرونة.
- **JWT (JSON Web Tokens):** لإدارة جلسات المستخدمين والمصادقة (Authentication) بشكل آمن.

### الخدمات الخارجية (Services):
- **Cloudinary:** لتخزين وإدارة الصور المرفقة مع البلاغات بشكل سحابي ومحسّن.
- **Resend:** خدمة لإرسال رسائل البريد الإلكتروني (تفعيل الحساب، استعادة كلمة المرور).

### إجراءات الأمان (Security Measures):
- **Helmet:** لضبط ترويسات HTTP (HTTP Headers) لحماية التطبيق من ثغرات الويب المعروفة.
- **Rate Limiting:** تحديد عدد الطلبات المسموح بها لكل مستخدم/IP لمنع هجمات الـ DDoS و Brute Force.
- **Input Sanitization:** تنظيف المدخلات لمنع هجمات XSS و NoSQL Injection.
- **CORS Configuration:** ضبط سياسات مشاركة الموارد عبر الأصول لضمان أن الـ Frontend فقط هو من يمكنه الاتصال بالـ Backend.
- **IP Blacklisting:** نظام لحظر عناوين IP المشبوهة أو الضارة.
- **Password Hashing:** تشفير كلمات المرور باستخدام `bcryptjs` قبل تخزينها.

## 4. قاعدة البيانات (Database Models)

تم تصميم هيكلة البيانات (Schema Design) لتكون مترابطة وفعالة:

1.  **User (المستخدم):**
    -   بيانات المستخدم (الاسم، البريد، كلمة المرور المشفرة).
    -   حالة الحساب (مفعل، محظور).
    -   الدور (مستخدم عادي، مشرف، مسؤول).

2.  **Report (البلاغ):**
    -   تفاصيل عملية الاحتيال.
    -   الصور والأدلة المرفقة.
    -   حالة البلاغ (قيد المراجعة، مقبول، مرفوض).
    -   يرتبط بالمستخدم الذي قام بالإبلاغ وبالمحتال.

3.  **Scammer (المحتال):**
    -   بيانات المحتال (الاسم، أرقام الهواتف، الحسابات البنكية، منصات التواصل).
    -   يرتبط بجميع البلاغات المقدمة ضده.
    -   يتم تجميع البيانات لتسهيل البحث عنه.

4.  **BannedIP:**
    -   قائمة بعناوين IP المحظورة من الوصول للمنصة.

---

## 5. دورة حياة البلاغ (Report Lifecycle)

1.  يقوم المستخدم بتقديم بلاغ جديد مع الأدلة.
2.  يتم رفع الصور إلى Cloudinary وتخزين الروابط.
3.  يتم تحليل البيانات ومقارنتها بقاعدة البيانات (Matching Service) للتحقق من وجود بلاغات سابقة لنفس المحتال.
4.  يظهر البلاغ للمشرفين في لوحة التحكم.
5.  يقوم المشرف بمراجعة البلاغ وقبوله أو رفضه.
6.  في حال القبول، تظهر بيانات المحتال في نتائج البحث العامة لتحذير الآخرين.

---

## 6. أدوات التطوير والإنتاج (DevOps & Tools)

- **ESLint & Prettier:** لضمان جودة الكود وتنسيقه الموحد.
- **Git:** لإدارة نسخ الكود.
- **Environment Variables (.env):** لإدارة المفاتيح السرية والإعدادات بشكل آمن بعيداً عن الكود المصدري.
